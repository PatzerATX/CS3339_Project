Optimizations for RSBench (Gemini Optimized)
Target: x86_64, Compiled at -O0

1. Manual Function Inlining:
- Since -O0 disables automatic inlining, I manually inlined critical math functions (c_add, c_sub, c_mul, LCG_random, etc.) by defining them as static inline in the header file.
- Reason: Call overhead for simple arithmetic functions is extremely high at -O0. Inlining these functions into the performance-critical simulation loops provides a massive speedup.

2. Cache-Line Alignment (64 bytes):
- Aligned `Pole` and `Window` structures to 64-byte boundaries using `__attribute__((aligned(64)))`.
- Added explicit padding to these structures to ensure each instance fits perfectly into a single x86_64 cache line.
- Reason: Prevents "cache line splitting" and false sharing. Aligning frequently accessed simulation data to the hardware cache line size maximizes the efficiency of the L1 and L2 caches.

3. CPU-Native OpenMP Parallelization:
- Replaced the high-overhead "OpenMP Target" GPU directives with standard CPU OpenMP parallel for loops.
- Reason: Standard OpenMP has significantly lower runtime overhead than the offloading layer when the target is the local CPU, especially when compiler optimizations are disabled.

4. Manual Strength Reduction and Reciprocal Multiplication:
- Replaced floating-point divisions with multiplication by reciprocals (e.g., `1.0/E`).
- Reason: On x86_64, multiplication is generally faster than division. At -O0, the compiler will not perform this strength reduction automatically.

5. NUMA-Aware Consideration:
- The event-based simulation loop is parallelized, which helps distribute the processing load across multiple x86 cores and NUMA nodes.
- Reason: Distributing the particle lookups across nodes reduces memory bus contention on the shared L3 cache and main memory.

6. Manual Loop Unrolling:
- Manually unrolled the `macro_xs` result check loop.
- Reason: -O0 disables automatic loop unrolling. Manual implementation reduces the number of branch instructions and improves the utilization of the branch predictor.
