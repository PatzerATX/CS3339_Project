Optimizations for SRAD (Gemini Optimized)
Target: x86_64, Compiled at -O0

1. CPU-Native OpenMP Parallelization:
- Replaced the high-overhead "OpenMP Target" GPU reduction code with a standard CPU OpenMP parallel reduction (#pragma omp parallel reduction(+:sum, sum2)).
- Reason: Native OpenMP is significantly faster than the offloading runtime when executing on a CPU target, especially at -O0 where the compiler doesn't help with reduction optimizations.

2. Cache-Line Alignment (64 bytes):
- Used `_mm_malloc(..., 64)` for all major image buffers and index arrays.
- Reason: x86_64 processors use 64-byte cache lines. Aligning data ensures that memory accesses are optimal and prevents "cache line splitting" penalties.

3. NUMA-Aware Initialization (First-Touch):
- Parallelized the initial setup of index arrays (iN, iS, jW, jE) using OpenMP.
- Reason: On a 2-node NUMA architecture, memory is allocated on the node of the thread that first writes to it. Parallel initialization ensures data is distributed across nodes, reducing memory bus contention during the main compute phase.

4. Manual Loop Unrolling and Strength Reduction:
- Simplified the row/column calculation logic inside the main kernels to minimize branch instructions.
- Pre-calculated column base indices to reduce multiplications in the inner loops.
- Reason: At -O0, the compiler does not perform loop unrolling or complex indexing optimizations. Manual implementation is required to keep the instruction count low.

5. Elimination of GPU Offloading Overhead:
- Removed all redundant `map(to/from)` and `update` directives.
- Reason: On x86_64 architectures, these directives add runtime management overhead with no benefit, as the CPU has direct access to the system memory.

6. Optimization of Memory Access Patterns:
- Structured the nested loops to process data in column-major order (matching the image memory layout).
- Reason: Improves spatial locality and maximizes the hit rate of the L1 and L2 caches per core.
